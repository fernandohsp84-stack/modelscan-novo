Guia Completo
Como o ModelScan funciona?
Entenda cada etapa do processo de catalogação inteligente — dos agentes de IA ao Model Card final. Simples, transparente e feito para apoiar você.
```
Jornada completa — passo a passo
1
Identificação do Modelo
Você começa informando os dados básicos: nome do modelo, área responsável e quem é o Model Owner. Também insere os contatos de e-mail para que as notificações cheguem automaticamente às pessoas certas após a conclusão. Não precisa saber tudo — informe o que souber. Os agentes completam o restante.
💡 Dica: o nome do modelo pode ser informal agora. A área de validação poderá padronizar na etapa de ratificação.
2
Indicação das Fontes
Aqui você indica onde o modelo está documentado. Pode ser um arquivo PDF ou Word com a especificação técnica, um notebook Jupyter (.ipynb), scripts Python ou R, links do SharePoint, repositórios Git, projetos Databricks ou ambientes Google Cloud. Você pode combinar quantas fontes quiser — o sistema aceita tudo.
📄 PDF / Word 🐍 Python / Jupyter 📊 R / RMarkdown ☁️ SharePoint ⚡ Databricks 🔵 Google Cloud 🗂️ Git / GitLab 🔗 Links externos
3
Pipeline de 5 Agentes de IA
Ao clicar em "Analisar", cinco agentes especializados entram em ação sequencialmente:
🗂 Agente de Ingestão — lê e parseia todos os arquivos e links enviados, convertendo o conteúdo bruto em fragmentos de texto processáveis.
🔍 Agente RAG (Extração) — usa Recuperação Aumentada por Geração para responder as perguntas do formulário de catalogação diretamente a partir das fontes. Ele "sabe" o que está nos documentos e extrai: finalidade, inputs, outputs, técnica, plataforma, uso regulatório, grau de autonomia decisória.
🏷 Agente Classificador — enquadra o modelo em uma das 5 famílias (Estatístico Clássico, Matemático/Financeiro, Especialista/Julgamental, Machine Learning ou IA Generativa/LLM) e identifica a técnica principal.
⚖️ Agente de Criticidade — aplica a matriz de 6 dimensões ponderadas (Impacto Financeiro, Exposição Regulatória, Autonomia Decisória, Opacidade, Abrangência e Maturidade) para calcular o nível de risco de N1 (Crítico) a N5 (Mínimo).
📋 Agente de Síntese — consolida tudo em um Model Card estruturado, gera a chave única do modelo e prepara as mensagens automáticas para os destinatários.
4
Model Card + Moderação do Owner
O Model Card completo é gerado automaticamente com uma chave única (ex: MDL-A3F9K2). Você revisa os campos preenchidos pela IA, corrige o que precisar e confirma ou solicita revisão. O tempo médio desta etapa é de 5 a 10 minutos — bem menos que qualquer formulário tradicional.
💡 Se a confiança da análise estiver abaixo de 60%, o sistema sinalizará os campos que precisam de mais atenção.
5
Mensageria Automática
Ao finalizar, dois e-mails são enviados automaticamente: um para o Model Owner com o resumo do card e os próximos passos; outro para a Área de Validação de Modelos com o card técnico completo, pontuação de criticidade, controles recomendados e escopo de validação — tudo pronto para priorização da fila.
6
Ratificação pela Área de Validação
Na aba "Área de Validação", os analistas da equipe revisam cada Model Card, podem corrigir qualquer campo — inclusive o nível de criticidade —, adicionar observações técnicas e registrar a ratificação. Outras unidades (Auditoria, Compliance, Risco Operacional) também podem manifestar-se diretamente na ficha.
Perguntas frequentes
Preciso ter documentação formal para usar o sistema? +
Não! O sistema foi pensado para funcionar com o que você já tem — pode ser um README no repositório, comentários no código, uma apresentação em PowerPoint ou qualquer coisa que descreva o modelo. Quanto mais informação, mais precisa a análise, mas você não precisa preparar nada especial.
O que é a chave única do modelo? +
Cada modelo catalogado recebe um identificador único (ex: MDL-A3F9K2) que serve como referência permanente no inventário institucional. Ele aparece no Model Card, nas notificações de e-mail e em todo o ciclo de validação e monitoramento.
O que acontece se eu discordar do nível de criticidade sugerido? +
Você pode solicitar revisão na etapa de moderação, adicionando uma observação. A Área de Validação terá acesso à sua justificativa e poderá corrigir o nível durante a ratificação. O nível final sempre é determinado pelo analista de validação.
Quanto tempo leva todo o processo? +
A análise pelos agentes leva entre 15 e 45 segundos. A moderação pelo Owner costuma levar de 5 a 10 minutos. O processo inteiro — da primeira tela ao card registrado — pode ser concluído em menos de 20 minutos para a grande maioria dos modelos.
Meus arquivos ficam salvos onde? +
Os arquivos enviados são lidos localmente no seu navegador e processados para extração de texto. O conteúdo textual é enviado à API da Anthropic para análise. O Model Card gerado é salvo no inventário da plataforma. Veja a aba "Onde ficam os arquivos" para o mapa completo de armazenamento.
```

```
🔒
Os arquivos enviados não são armazenados permanentemente pela aplicação — apenas lidos para extração de texto durante a análise. O que fica salvo é o Model Card estruturado, não os documentos originais.
📋
Model Cards (fichas de catalogação)
/inventario/model-cards/{ID_MODELO}/
Cada Model Card gerado é salvo com sua chave única. Contém todos os metadados: família, técnica, pontuação de criticidade, controles recomendados e histórico de moderação.
📧
Notificações e Mensagens
/inventario/mensageria/{ID_MODELO}/
Cópia de cada e-mail enviado (owner + validação) fica registrada aqui com timestamp, destinatários e conteúdo para fins de auditoria e rastreabilidade.
🔍
Histórico de Validação
/inventario/validacoes/{ID_MODELO}/
Registro de todas as interações da Área de Validação: ratificações, correções, manifestações de outras unidades e histórico de alterações no nível de criticidade.
📊
Base do Inventário (BI)
/inventario/consolidado/inventario.json
Arquivo consolidado com todos os modelos ativos, usado para alimentar o Painel BI. Atualizado automaticamente a cada nova catalogação ou ratificação.
🗃️
Arquivos originais (referência)
/inventario/fontes/{ID_MODELO}/
Os arquivos enviados pelo Owner ficam armazenados aqui para consulta futura pela Área de Validação. Acesso restrito por perfil. Retenção: 5 anos ou conforme política institucional.
🔑
Chaves únicas de modelo
/inventario/registro-ids/
Registro central de todos os identificadores gerados. Garante unicidade e rastreabilidade de cada modelo ao longo de todo o seu ciclo de vida (criação, validação, monitoramento, descomissionamento).
```
